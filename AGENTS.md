# FileUpAPI_AWS Agent Rules

このファイルは `FileUpAPI_AWS` での作業に対する強制ルールです。  
親ワークスペースの `AGENTS.md` に加えて、以下を必須とします。

## 実装前ゲート（必須）

- コード変更・設定変更・ドキュメント変更を含む**すべての編集作業**の前に、必ず `implement-workflow` を適用する。
- 編集前に必ず次を提示する。
  - `Implementation Plan`
  - `Test Perspectives`
- ユーザーの明示承認を得るまで、`apply_patch` やファイル更新コマンドを実行してはいけない。

## レビューゲート（必須）

- 実装完了後は必ず `critical-review` を実施する。
- 指摘が出た場合、修正または未対応理由を明記するまで完了扱いにしない。

## 例外ルール

- 緊急修正・軽微修正・文言修正でも、上記ゲートを省略してはいけない。
